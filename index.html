Tips on writing commit messages

Bad!


this is what i did 1) this 2) that 3) this and that; nice i like what i did

Write like email


first line: short summary, like "subject"
<second line empty>
third line onwards: more details, if required; like "body".

Final words

VCS is a tool

  • (or set of tools)

  • up to you how to use it

Logical, atomics commits

  • for checkpointing to be useful

  • granularity is arbitrary

Distributed = speed

  • but comes at a price

  • only one copy of your "unpushed" work

Not a backup solution

  • un-committed changes can be lost
